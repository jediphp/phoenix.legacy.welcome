# Prompt на внедрение адаптивности и нормализацию типографики

## Контекст
Проект — презентационный Next.js/Tailwind-интерфейс со слайдами для двух сценариев (`author` и `org`).
Задача: провести рефакторинг адаптивности без изменения контента слайдов, чтобы убрать «рассыпание» композиции на разных разрешениях, устранить слишком мелкие размеры текста и заменить жёсткие величины на масштабируемую систему.

---

## Что уже выявлено в кодовой базе (обязательно учесть)

1. **Есть большое количество очень мелких шрифтов (`text-[6px]…text-[11px]`)**, особенно в author-слайдах.
2. **Много жёстко заданных размеров в `px`** (ширины/высоты, маркеры, декоративные элементы, фиксированные min-height), что ведёт к ломанию композиции на низкой высоте экрана.
3. **`author-flow.css` уже содержит workaround через принудительные `!important`-правила**, подменяющие мелкие `text-[Npx]`.
4. **Несогласованность подходов**: в org-слайдах заметно больше `clamp(...)` и `vh/vw`, в author-слайдах чаще точечные размеры и ручные исключения.
5. **Встречаются фиксированные «световые»/декоративные блоки 800px**, которые не масштабируются от viewport.

---

## Цель рефакторинга

Сделать полностью управляемую, предсказуемую систему адаптивной верстки:

- единые responsive-токены,
- единая типографическая шкала,
- отказ от хаотичных `text-[Npx]` и локальных костылей,
- стабильное отображение на `320px → 2560px` по ширине и `568px → 1440px` по высоте,
- отсутствие горизонтального скролла и текстовых наложений.

---

## Жёсткие требования к реализации

1. **Контент и смыслы не менять** (тексты, последовательность слайдов, логика сценариев).
2. **Не переписывать всё заново** — рефакторить по слоям (tokens → layout primitives → slides).
3. **Сохранить текущую визуальную стилистику** (темы, палитра, атмосфера), но привести размеры к системе.
4. **Минимальный читаемый размер шрифта**:
   - desktop: не ниже `12px` (эквивалент),
   - mobile: не ниже `11px` (эквивалент),
   - служебные подписи допускаются ниже только при явном обосновании.
5. **Убрать глобальные `!important`-костыли для шрифта** после перевода слайдов на системные токены.

## План работ (пошагово)

### Этап 1. Аудит и карта проблем

- Сформировать таблицу по всем слайдам (`AuthorSlide*`, `OrgSlide*`, `Home`, `PresentationFlow`):
  - где `text-[Npx]`,
  - где жёсткие `w/h/min/max` в `px`,
  - где inline style с `px`,
  - где переполнение/обрезка контента по высоте.
- Разделить проблемы на категории:
  - **Типографика**, **Сетки/контейнеры**, **Декор**, **Интерактивные элементы**.

### Этап 2. Дизайн-токены адаптивности

Создать/расширить слой токенов в стилях:

- fluid-типографика (`--fs-caption`, `--fs-body-sm`, `--fs-body`, `--fs-title-sm`, `--fs-title`, `--fs-display` через `clamp()`),
- интервалы (`--space-1...--space-8`) с viewport-aware диапазонами,
- радиусы/толщины линий/иконки,
- ограничители ширины контента (`--content-max`, `--content-wide`).

Результат: базовые размеры управляются из одного места, а не вручную в каждом компоненте.

### Этап 3. Базовые layout-примитивы

Ввести/привести к единому виду классы-обертки:

- `slide-shell` (высота, паддинги, safe-areas, overflow-policy),
- `slide-header`, `slide-body`, `slide-footer`,
- `meta-label`, `caption`, `micro-label` — вместо прямых `text-[7px]` и т.п.

Обязательное правило: **никаких новых локальных `text-[Npx]`** в слайдах.

### Этап 4. Рефакторинг author-слайдов (приоритет)

- Последовательно пройти `AuthorSlideOne` → `AuthorSlideFourteen`.
- Заменить мелкие размеры текста на токены.
- Убрать критичные фиксированные `min-height`/`w-[Npx]`/`h-[Npx]`, где они ломают layout.
- Для сложных сцен (heatmap, markers, callouts):
  - перевести размеры маркеров/бейджей/подписей в clamp/rem,
  - добавить адаптивные ограничения для контейнера,
  - предусмотреть режим упрощения на низкой высоте viewport (скрытие вторичных подписей, но без потери ключевого смысла).

### Этап 5. Нормализация org-слайдов

- Оставить сильные места (там уже много `clamp`),
- выровнять шкалу типографики и отступов с общей системой,
- унифицировать подписи прогресса/метки внизу слайдов.

### Этап 6. Global/flow-уровень

- `PresentationFlow` и `Home`:
  - адаптировать большие fixed glow-блоки (`800px`) к fluid-диапазону,
  - проверить поведение при `100svh` и малой высоте,
  - гарантировать, что snap/scroll не ломает доступность контента.
- Проверить глобальные CSS-файлы:
  - убрать временные правила, которые маскируют проблему, а не решают её.

### Этап 7. Валидация и регрессия

Проверить минимум на разрешениях:

- Mobile: `320x568`, `360x800`, `390x844`
- Tablet: `768x1024`, `1024x768`
- Laptop/Desktop: `1366x768`, `1440x900`, `1920x1080`, `2560x1440`

Для каждого сценария (`/author`, `/org`) зафиксировать:

- нет ли обрезаний текста,
- нет ли наложений,
- нет ли горизонтального скролла,
- читаемость микро-текста,
- стабильность анимаций.

## Definition of Done (DoD)

Считать задачу завершённой только если:

1. В коде слайдов нет россыпи `text-[6px]..text-[10px]` как основного метода типографики.
2. Удалены/минимизированы `!important`-патчи для forced font-size в `author-flow.css`.
3. Ключевые блоки в сложных author-сценах не выходят за контейнер при малой высоте экрана.
4. На целевых viewport отсутствуют критические визуальные дефекты.
5. Добавлен краткий технический отчёт:
   - что было найдено,
   - какие токены введены,
   - какие компоненты рефакторились,
   - какие компромиссы приняты.
